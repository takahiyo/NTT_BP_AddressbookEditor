# デバッグログ：電話番号加工ロジックの柔軟性向上

## 状況
- 現在のロジックは市外局番が3桁であることを前提とした「7桁なら補完」という固定ルールになっている。
- 全国（2〜5桁の市外局番）に対応できるよう、ターゲット桁数ベースの動的なロジックが必要。

## 調査工程
1. [x] 現状の `phone-processor.js` の実装を確認。
2. [x] 市外局番の桁数に応じた補完パターンの整理。
3. [ ] 実装の修正とエッジケースの検討。

## 対応案
- 設定された `cityCode` の桁数を取得。
- `10桁（固定電話） - cityCodeの桁数` と入力桁数が一致する場合を補完対象とする。
- すでに `0` から始まっている場合は、重複付与を避ける。
- 11桁（携帯）はそのまま（先頭0がなければ付与）処理する。
