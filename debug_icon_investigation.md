# アイコン番号の範囲に関する調査記録

## 調査目的
現在の3機種（A1, ZX2SM, ZX-L）におけるアイコン番号の定義状況と範囲を把握する。

## 調査結果

| 機種 | 定義ファイル | アイコン番号の範囲 | デフォルト値 |
| :--- | :--- | :--- | :--- |
| **A1 (A1Std/Pro)** | `a1.js` | `1` ～ `9` | `1` |
| **ZX-L** | `zxl.js` | `16, 17, 20, 21, 24, 25, 28, 52, 23` | `16` |
| **ZX2SM** | `zx2sm.js` | `1` ～ `8`（バリデーターのデフォルト値） | なし |

### 補足事項
- **バリデーションロジック**: `validator.js` にて、機種仕様に `iconRange` が定義されていない場合はデフォルトとして `1-8` が適用される仕組みになっています。
- **自動補正**: `phone-processor.js` の `processAllPhoneNumbers` 関数内において、アイコン番号が未入力または `0` の場合に `1` に自動補正するロジックが存在します。
- **ZX2SMの型定義**: `zx2sm.js` ではアイコン番号が `type: 'text'` となっており、他の2機種（`type: 'number'`）と不整合がある可能性があります。

## 今後の対応案
- ZX2SM の機種定義（`zx2sm.js`）にも `iconRange` を明示的に追加し、SSOT（Single Source of Truth）を強化する。
- ZX2SM のアイコン番号の型を `number` に統一することを検討する。
