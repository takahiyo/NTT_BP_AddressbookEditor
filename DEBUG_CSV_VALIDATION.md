# デバッグログ: CSVファイル形式チェックの強化

## 背景
明らかに弊社電話機のCSV形式ではないファイルが読み込まれた際、エラーとして処理を停止させたい。

## 工程
- [x] 現状のCSV読み込みおよび自動判別ロジックの調査
  - `src/js/services/csv-parser.js` の `detectSpecFromCSV` で判別
  - `parseCSVFile` でパース
- [x] 判定基準（ヘッダー、列数、エンコーディング等）の策定
  - `detectSpecFromCSV` が `null` を返す場合は非対応とみなす
  - 列数チェックの更なる厳格化
- [/] エラー処理の実装
- [ ] 動作確認

## 調査結果
`detectSpecFromCSV` は全登録済み仕様をループして判別している。
これが `null` を返すということは、どの既知の形式（列数やシグネチャ）にも当てはまらないことを意味する。
これを `parseCSVFile` 内でフックし、エラーを投げれば `app.js` の `try-catch` で捕捉されトースト表示される。
